## 1. 目标与边界

本清单用于把现有高保真原型图沉淀为可测量、可复用、可验收的开发前视觉设计资产，使后续前端实现能够最大程度还原已确认的视觉效果。

目标资产覆盖：

- 15 个 MVP 页面与状态的 Dark 基线。
- 15 个页面与状态的 Light 基线。
- 共享 Design Tokens 与组件目录。
- 关键交互、动效、无障碍和失败状态。
- Profile、Radio、Detail、Management、Settings 五类布局的响应式代表页。
- 可追溯的视觉裁决记录、基线截图和前端交接映射。

本阶段不得：

- 创建或修改 `apps/`、`packages/design-tokens/` 或其他产品源码。
- 初始化 monorepo、包管理器、生产依赖或应用运行配置。
- 将设计原型中的静态数据描述为真实服务、接口或产品实现。
- 把 Figma、截图或现有 PNG 设为与 HTML 并列的权威主源。
- 将本清单与后续产品开发任务混合记录。

## 2. 资产权威关系

当前 15 张 [参考图](../references/) 是已确认的视觉参考，实际尺寸均为 `971 × 1619px`。目标 HTML 基线统一使用 [视觉设计系统规范](../design.md) 定义的 `960 × 1600px` 画布。

完成视觉冻结前：

1. 产品内容、状态和操作路径以 [产品需求文档](../../docs/prd.md) 与 [用户流程](../../docs/user-flow.md) 为准。
2. 明确的布局、尺寸、颜色、组件和无障碍规则以 [视觉设计系统规范](../design.md) 为准。
3. 文档未规定的视觉细节以对应 PNG 为参考。

完成 `VDA-16` 后：

- 静态 HTML / CSS / JavaScript 成为开发前视觉设计资产的唯一主源。
- 基线截图用于视觉对比，不反向定义组件尺寸。
- Figma 是从已验收 HTML 派生的协作镜像，不允许单独修改后成为新事实源。

## 3. 视觉差异裁决

### 3.1 权威顺序

1. 产品语义和流程冲突：`docs/prd.md`、`docs/user-flow.md` 胜出。
2. 明确的布局、尺寸、组件族和无障碍规则：`design/design.md` 胜出。
3. 文档未规定的视觉细节：对应 PNG 胜出。
4. PNG 之间存在冲突：选择跨页面最一致、最清晰且无生图畸变的样本，沉淀为共享组件。
5. PNG 的方案明显更优但违反规范：提交变更提案；确认后先更新 `design/design.md`，再修改 HTML。

不得为实现方便静默折中，也不得对多张 PNG 的尺寸取平均值。

### 3.2 归一化与容差

- 对比前将 `971 × 1619px` PNG 归一为 `960 × 1600px`，整体导出缩放不记为设计差异。
- 使用半透明叠图、边缘图和关键锚点测量检查差异。
- PNG 参考允许不超过 `4px` 的生图漂移。
- HTML 主源中的规范坐标和组件尺寸必须精确实现。
- 字体抗锯齿、随机波形和轻微图片裁切差异不做逐像素追随。
- 共享组件只能保留一个明确规格，页面不得维护局部变体规避裁决。

### 3.3 确认级别

| 级别 | 范围 | 处理方式 |
|---|---|---|
| A：必须确认 | Logo / 字标、页面骨架、内容层级、主组件结构、主题气质、交互语义、规范变更 | 提交局部对比图、推荐方案、收益与影响，等待用户确认 |
| B：批量确认 | 明显影响整页观感的密度、材质、排版比例和组件统一方案 | 按页面族集中提交，不逐项打断 |
| C：自动裁决 | 普通间距、弱阴影、分割线、未规定的微观细节和 `4px` 内生图漂移 | 按规范和组件一致性处理，在页面族验收时统一展示 |

有效差异写入目标文件 `design/assets/reports/visual-decisions.md`，格式如下：

```text
编号 / 页面与区域 / 差异类别 / PNG 表现 / 规范要求
严重级别 / 推荐裁决 / 影响页面 / 最终决定 / 证据截图
```

已经裁决的问题不得在后续页面中选择另一套方案。若决定改变产品行为、用户流程或 UI 规范，必须先更新对应权威文档。

### 3.4 视觉确认门

1. **共享组件门**：`VDA-02` 完成后，确认 Tokens、整体材质和共享组件标准；2026-07-13 尺度补正确认点已通过。
2. **核心体验门**：01–08、共享组件的新尺度与 Detail Sheet 双态已于 2026-07-13 完成自动验收；两态共享固定骨架，核心体验确认门已通过。
3. **整体冻结门**：`VDA-14` 与 `VDA-15` 完成后，由 `VDA-16` 确认 15 页、Light 主题、响应式代表页、状态映射与开发验收清单；2026-07-14 已通过。

确认门未通过时，后续依赖任务不得标记为已完成。

## 4. 任务管理规则

任务状态只使用：

- `待开始`
- `进行中`
- `待验收`
- `已完成`
- `阻塞`

执行规则：

- 每个 `VDA` 任务原则上占用一个独立 Codex 会话。
- 每次会话仍须按仓库规定读取 `README.md`、`AGENTS.md`、`AI_RULES.md` 和 `context.md`。
- 页面任务只加载对应 PRD、流程、设计规范章节和直接相关参考图。
- 必须共同校准的状态页面保留在同一任务中，不拆成独立会话。
- 每个任务只修改自身交付物和必要的任务状态，不夹带产品代码或无关重构。
- 只有交付物存在、验收项通过且依赖确认门满足后，任务才能标记为 `已完成`。

## 7. 分任务验收

### VDA-00｜资产盘点与差距审计

- 清点 01–15 文件、尺寸、页面状态和对应权威章节。
- 将原图归一化后检查画布、内容列、固定 slot、组件比例和跨页面一致性。
- 为 A / B 级差异建立待裁决条目；C 级差异记录自动裁决规则。
- 不修改原始参考图。

### VDA-01｜HTML 原型骨架

- 使用静态 HTML / CSS / JavaScript，不引入产品框架和包管理器。
- 支持按页面、主题和预设 viewport 定位唯一状态。
- fixture 固定且不模拟真实 Backend、Provider 或数据库。
- 明确设计预览与产品可运行状态无关。

### VDA-02｜Tokens 与组件目录

- Tokens 与 `design/design.md` 一一对应，页面不得复制独立色值和尺寸体系。
- 展示品牌区、导航、按钮、输入、卡片、状态、媒体容器、排版和 Focus 状态。
- 同类组件只保留一个基准规格。
- A / B 级组件差异完成裁决并通过共享组件确认门。

### VDA-03 至 VDA-09｜页面族

- 页面内容、状态词和恢复入口与 PRD、用户流程一致。
- 几何尺寸和组件用法符合设计规范。
- 页面族内的共享组件和固定区域复用同一实现。
- 不通过页面私有样式掩盖组件不一致。
- VDA-05 继续使用 Detail Sheet 全屏骨架并继承共享排版与固定比例控件；VDA-06–08 使用 Management 内容列，VDA-09 使用 Settings 内容列；所有一级页面继承同一顶部工具与底部导航 token。
- 后续页面不得引用 VDA-00 历史记录中的小规格，也不得复制 rail、顶部工具、表单、媒体或导航尺寸；精确数值统一从 `design/design.md`、`design/prompt.md` 与原型 `tokens.css` 读取。
- 每个页面族完成 Dark 基线审查并关闭该范围内的 A / B 级差异。

### VDA-10 至 VDA-11｜Light 主题

- 15 个页面均使用 Light 主题变量渲染。
- 不做 Dark 主题的机械反色。
- 检查材料层级、边框、Focus、状态色、浅色节目面和文字对比度。
- 页面结构与 Dark 版本保持一致。

### VDA-12｜响应式代表页

- 使用 `390 × 844`、`834 × 1194`、`1440 × 1200` 三组 viewport。
- 03、05、08、10、14 分别代表 Profile、Radio、Detail、Management、Settings 布局族。
- 验证滚动、固定区域、安全区、长文案和 44px 最小命中区。
- 记录同族其他页面继承的布局规则，不复制整套页面变体。

### VDA-13｜交互与无障碍

- 覆盖 hover、active、focus、disabled、loading、error 和键盘路径。
- 验证可见 Focus、WCAG AA、`aria-live`、200% zoom 与 Reduce Motion。
- Detail Sheet 开合、页面切换和波形状态符合动效规范。
- 状态不得只依赖颜色表达。

### VDA-14｜截图基线与视觉 QA

- Dark 与 Light 各输出 15 张 `960 × 1600px` 基线。
- 输出响应式代表页基线。
- 检查文字准确性、裁切、固定比例、跨状态几何、对比度和恢复入口。
- 所有 A / B 级差异必须关闭，C 级裁决必须可追溯。

### VDA-15｜Figma 派生镜像

- 新建 `Koradio · Visual Baseline · MVP` Figma 文件。
- 页面 Frame、共享组件和变量映射来自已验收 HTML。
- 记录生成日期、HTML 版本和已知转换限制。
- Figma 中发现的问题必须回到 HTML 主源修正后重新同步。

### VDA-16｜视觉冻结与开发交接

- 建立页面、状态、组件、token、fixture 和基线截图映射。
- 形成前端实现的视觉验收清单。
- 通过整体冻结门并记录冻结版本。
- 后续视觉变更先更新权威文档和 HTML 主源，再生成截图与 Figma 派生物。

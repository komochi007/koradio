| 问题 | 直接答案 |
|---|---|
| 当前工程阶段 | Documentation-first；产品工程尚未开始 |
| 当前已完成任务 | `S0-01`、`S0-02`、`S0-03`、`S0-04`、`S0-05`、`S0-07` |
| 当前活动任务 | `S0-06` Provider 技术与发布可行性验证阻塞；Browser Audio Engine 已确认保留，网易云官方 Browser 播放资源能力与授权尚无证据 |
| 初始下一任务 | 取得网易云官方 Browser 播放能力，或裁决替换音乐 Provider；S0 阶段门关闭前不启动 S1 |
| 首轮外部测试条件 | `S5-04`、`S6-05`、`S7-05` 全部为 `已完成` |
| 当前交付与后续发布 | 先做项目所有者本机 Personal Local Preview；公开下载获再次授权后才进入签名公证直发 |
| 状态事实源 | 只修改本文件；GitHub Issue/Project 只写入“外部引用” |

AI 执行任务时按固定顺序操作：

1. 在下表筛选状态为 `待开始` 且所有依赖为 `已完成` 的任务。
2. 读取该行“输入”和对应权威文档，重新检查真实文件树与 Git 状态。
3. 确认阻塞条件未发生，再把状态改为 `进行中`、填写负责人，并在同一修改中同步“状态摘要”。
4. 只实施“范围边界”允许的内容，产生“交付物 / 计划路径”。
5. 执行“验证方式”；交付物和自检完成后改为 `待验收`。
6. 验收标准全部成立并记录证据后改为 `已完成`，再次同步“状态摘要”；阶段末同步 `roadmap.md`。

## 2. 表格字段规则

| 字段 | AI 解析规则 |
|---|---|
| ID / 任务 | ID 全局唯一；任务名描述可验收结果 |
| 状态 | 仅允许 `待开始`、`进行中`、`待验收`、`已完成`、`阻塞` |
| 优先级 | `Critical` > `High` > `Medium` > `Low`；不能绕过依赖 |
| 依赖 | 仅引用本表任务 ID；所有依赖完成后任务才 Ready |
| 输入 | 开始前必须读取或取得的文档、决策、环境或凭据 |
| 范围边界 | 同时定义必须做和不得做的内容 |
| 验收标准 | 可观察的 Done 条件，不接受主观“基本完成” |
| 交付物 / 计划路径 | 任务应产出的文件、代码、报告或安装包；完成前均是计划路径 |
| 验证方式 | 必须实际执行的测试、检查或人工验收 |
| 负责人 / 外部引用 | 未分配写 `待分配`；GitHub/ADR/PR 只作反向引用 |
| 阻塞条件 | 只有条件真实发生且无安全替代路径时才标记 `阻塞` |

表格是每个任务的最小执行合同：未写入“范围边界”的工作默认不在任务范围内；“验收标准”必须与“输入”引用的权威文档同时满足，不能用表格中的压缩描述降低原有标准；阻塞解除后必须记录解除证据。

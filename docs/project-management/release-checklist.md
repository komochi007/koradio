## 1. 使用规则

- 本清单是发布硬门，不是提醒列表。标记完成时必须附可追溯证据，不能只写“已确认”。
- `不适用` 必须说明理由和批准人；安全、数据完整性、签名公证、回滚和核心 E2E 不允许标记为不适用。
- External Beta 只有在 S5、S6、S7 阶段门全部通过后才可开始。
- RC 冻结后只允许修复发布阻断问题；任何二进制变化都必须生成新 RC 并重跑适用检查。
- 不在本文件写入密钥、token、Apple 凭据、用户路径、原始 Provider 正文或未脱敏日志。

每次发布先复制以下记录头到对应发布记录：

```text
源码提交：
构建流水线：
安装包文件名：
SHA-256：
签名身份摘要：
公证记录：
测试负责人：
发布负责人：
回滚负责人：
Go/No-Go 时间：
```

## 3. Release Candidate 冻结

### 缺陷与变更控制

- [ ] 所有 Beta 反馈已分级并有处理结论。
- [ ] 无未解决 Blocker 或 Critical 缺陷。
- [ ] High 缺陷已修复；未修复项均有风险、缓解措施、责任人和发布豁免。
- [ ] 每个 bug fix 都有修复前失败、修复后通过的 regression test。
- [ ] 新功能和范围外改进已移至后续版本，没有混入 RC。
- [ ] 最后一次代码变更后完整质量流水线、安装生命周期和关键人工回归已重新执行。

### 版本与产物冻结

- [ ] RC 版本号、源码提交、依赖锁文件、构建流水线和安装包一一对应。
- [ ] RC 安装包已签名、公证、staple 并通过系统验证。
- [ ] SHA-256 已生成并由独立下载文件复核。
- [ ] 构建和公证日志已脱敏，凭据未进入日志、仓库或产物。
- [ ] RC 后没有未提交、未构建或未测试的产品改动。
- [ ] 若 RC 发生任何二进制变化，已使用新 RC 编号并重跑本节全部检查。

### 全量回归

- [ ] 全新安装和至少一个旧 Beta 升级到 RC 均通过。
- [ ] 升级后 Profile、Program、Feedback、TasteOverrides、Preferences 和 checkpoint 可读取。
- [ ] 首次配置和三类 Provider 健康检查行为正确，TTS 可选边界未改变。
- [ ] 场景生成期间旧节目继续播放，失败保持旧节目，成功后原子切换。
- [ ] 长时播放、暂停、seek、切歌、Detail Sheet、歌词/串讲跟随和媒体失败恢复通过。
- [ ] Library 搜索/导入、Taste 编辑、Programs 历史/复用/收藏通过。
- [ ] 多标签接管、Profile 切换、完全离线和数据迁移回滚通过。

## 4. v1.0 Go/No-Go

### 产品与质量签署

- [ ] 产品范围与 PRD 一致，所有验收例外有明确记录。
- [ ] 架构、数据、安全和模块 owner 与实现一致。
- [ ] VDA-17 视觉回归和无障碍验收通过，未出现未经批准的平行视觉规范。
- [ ] 质量负责人确认全部硬门通过且无跳过关键检查。
- [ ] 发布负责人确认只有冻结 RC 会被发布。

### 法务、隐私与用户文档

- [ ] `PRIVACY.md` 准确说明本地数据、外部 Provider 和诊断边界。
- [ ] `SECURITY.md` 提供受支持版本、安全报告渠道和敏感信息注意事项。
- [ ] `THIRD_PARTY_NOTICES.md` 与最终依赖和捆绑运行时一致。
- [ ] Provider 公开发布条款和音乐内容边界已有结论，无未处理阻断风险。
- [ ] 是否创建 `LICENSE` 已有明确授权决策；未授权时不得暗示仓库为开源软件。
- [ ] 安装、首次配置、密钥、数据目录、升级、回滚、卸载和诊断文档已按公开包实测。

### 发布与回滚准备

- [ ] Release notes、CHANGELOG、已知问题、支持系统范围和校验和已准备。
- [ ] GitHub Release 或项目下载入口已建立草稿，尚未提前公开。
- [ ] 公开安装包文件名、版本和 SHA-256 与冻结 RC 完全一致。
- [ ] 回滚手册已演练，明确何时暂停下载、撤回链接或发布 Hotfix。
- [ ] 发布后冒烟使用干净 Mac、公开下载链接和受控 Provider 配置。
- [ ] 发布、支持、回滚和安全响应负责人均已确认可用。

### Go/No-Go 结论

- [ ] 所有不允许豁免的检查均通过。
- [ ] 所有豁免包含风险、用户影响、缓解措施、负责人和后续任务。
- [ ] Go/No-Go 记录包含版本、产物、证据链接、参与者、时间和最终结论。
- [ ] 结论为 `Go` 后才允许执行公开发布；结论为 `No-Go` 时发布入口保持关闭。

## 5. 公开发布执行

- [ ] 从冻结 RC 直接发布，未在发布步骤重新构建或替换二进制。
- [ ] GitHub Release 或下载页面包含安装包、SHA-256、release notes、已知问题和安装说明。
- [ ] 发布页面不包含内部路径、token、凭据、原始日志或 Provider 敏感正文。
- [ ] 从公开入口重新下载安装包，校验 SHA-256、签名、公证、版本和文件大小。
- [ ] 在干净 Mac 完成公开下载 → 安装 → 首次启动 → 配置 → 生成 → 播放 → 反馈冒烟。
- [ ] 完成一次公开版本手动升级和回滚演练。
- [ ] 确认公开链接、支持渠道、安全报告入口和隐私文档均可访问。
- [ ] 发布记录写入最终 URL、时间、版本、校验和和冒烟结果。

## 6. 发布阻断与回滚条件

以下任一情况发生时必须停止或撤回分发：

- [ ] 安装包签名、公证、校验和或版本验证失败。
- [ ] 首次启动、核心配置、节目生成或播放在支持环境普遍失败。
- [ ] 升级或迁移存在数据损坏、丢失或无法回滚风险。
- [ ] 密钥、token、敏感路径、原始 Provider 正文或用户数据被暴露。
- [ ] 出现可远程利用或高影响的 Critical 安全问题。
- [ ] 发布产物与冻结 RC、源码提交或 release notes 不一致。

执行回滚时：

- [ ] 立即暂停下载入口并记录发现时间、影响版本和已知范围。
- [ ] 保留故障证据但先完成脱敏，不删除用户数据、旧数据目录或备份。
- [ ] 告知用户停止升级或使用受影响能力，并提供安全的临时措施。
- [ ] 按回滚手册恢复上一安全版本或发布新 Hotfix 版本；不得覆盖同版本二进制。
- [ ] 为问题添加 regression test，重新执行签名、公证、全量回归和 Go/No-Go。
- [ ] 恢复分发前再次从公开入口完成下载、安装、升级和回滚冒烟。

## 7. 稳定期与 Hotfix

- [ ] 发布后反馈按 Blocker/Critical/High/Medium/Low 持续分级。
- [ ] Blocker/Critical 立即进入热修复流程，并评估是否暂停分发。
- [ ] Hotfix 只包含缺陷或安全修复，不混入新功能和无关重构。
- [ ] 每个 Hotfix 使用新版本号、独立 release notes、签名、公证和校验和。
- [ ] Hotfix 重跑受影响测试、完整核心 E2E、安装/升级/回滚和安全检查。
- [ ] 稳定期退出时无未处理 Blocker/Critical，High 均有明确结论。
- [ ] v1.0 复盘记录事实、根因、有效控制、失效控制、后续任务和负责人。
- [ ] Windows、自动更新、Mac App Store 和新功能只进入后续 roadmap，不回写为 v1.0 已完成内容。

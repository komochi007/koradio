## 6. 关键异常分支清单

```text
[本地服务未连接]
  -> 显示独立“本地服务未连接”异常页
  -> 已打开或缓存的 PWA 展示 ○ OFFLINE、重试、只读 Settings、启动说明和脱敏诊断
  -> 配置、密钥、测试和迁移控件全部禁用
  -> 连接成功后返回 Radio

[阻断错误]
  -> 在相关区块内联显示
  -> 提供恢复入口：重试 / 前往 Settings / 修改输入 / 降级继续
  -> 不只依赖 Toast

[未配置核心服务]
  -> 场景发送被阻止
  -> 用户看到缺少 Codex 配置
  -> 跳转 Settings
  -> 保存并测试后重试

[Apple 系统 TTS 不可用]
  -> 不阻止场景发送
  -> Program 保留文字 DJ segment
  -> PlaybackTimeline 不创建伪音频 item

[Codex 规划失败]
  -> 保留用户输入
  -> 显示“点击重试”
  -> 用户重试或修改场景

[网易云无结果]
  -> 系统换关键词重试 2 次
  -> 仍无结果时提示换描述
  -> 不创建空节目

[TTS 失败]
  -> 保留文字 DJ
  -> 队列继续播放歌曲
  -> DJ 状态显示“文字串讲”

[TTS 无分句时间戳]
  -> Detail Sheet 保留串讲文本
  -> 按文本长度估算当前句高亮
  -> 播放不受影响

[歌词缺失]
  -> Detail Sheet 显示“暂无歌词”
  -> 波形和播放不受影响

[主题切换失败]
  -> Settings 回滚到上一次主题
  -> Radio 和 Detail Sheet 保持原主题
  -> 提示“主题保存失败”

[DJ 语言切换]
  -> Settings 保存 zh-CN 或 en-GB
  -> 下一次 Codex 规划使用新语言
  -> 当前已生成节目不强制重写串讲

[播放失败]
  -> 当前曲标记不可播放
  -> 自动尝试下一首
  -> 全部失败时提示重新生成

[反馈保存失败]
  -> 播放不中断
  -> UI 状态回滚或提示未保存
  -> 用户可稍后重试

[Reduce Motion]
  -> 关闭持续波形和大部分转场动画
  -> 保留 PLAYING / SPEAKING / THINKING 等文字状态
  -> 播放和流程不受影响
```

## 7. 开发前校准验收矩阵

| 场景 | 前置条件 | 关键动作 | 必须结果 |
|------|----------|----------|----------|
| 首次启动 | 无 bootstrap 与数据目录 | 启动 Local Service | 自动选择并创建可写 OS 应用数据目录，不要求用户手填路径 |
| TTS 缺失 | Codex、网易云可用，原生 helper 缺失或没有匹配语言的标准系统语音 | 提交合法场景 | 生成歌曲和文字 DJ；不创建伪音频项，不阻止播放 |
| 完全离线 | Local Service 不可达，PWA 已打开或缓存 | 进入 Settings | 仅只读展示启动说明、脱敏状态和重试；所有配置、测试与迁移控件禁用 |
| 迁移成功 | 目标目录为空且可写 | 发起迁移 | 暂停任务/播放并存 checkpoint，备份复制校验，原子切换 bootstrap 后重启；旧数据保留 |
| 迁移回滚 | 任一迁移阶段失败 | 等待任务结束 | bootstrap 继续指向旧目录，恢复旧数据运行，不自动删除任何目录 |
| 播放中生成 | 旧节目正在播放 | 提交新场景 | 旧节目继续；失败保持不变；成功提交后保存旧 checkpoint、停止旧时间线并原子切换 |
| Profile 切换 | 旧 Profile 正在生成或播放 | 选择新 Profile | 取消旧任务并丢弃迟到事件，保存并停止旧播放，再加载新 Profile |
| 双标签接管 | 标签 A 持有租约 | 标签 B 申请接管 | A 先存 checkpoint 并停止；B 获得新 epoch 后才能播放；续约 `2s`、过期 `5s` |
| 反馈撤销 | 已存在喜欢、不喜欢或节目收藏 | 执行撤销 | 追加对应 removed 事件，不覆盖旧事件，界面由最新有效事件投影 |
| Taste 重建 | 已有自动投影与人工规则 | 回放全部反馈事件 | 可重建 `TasteProjection`，`TasteOverrides` 完整保留并优先合并 |
